```
My Red Team capability stack:

  Full kill chain documentation with MITRE ATT&CK mapping
  Detection gap analysis

Professional note:
  All red team activities conducted under explicit written
  authorisation with defined rules of engagement.
  Purpose: improve detection capability, not cause harm.
```

## 🕵️ Dark Web & Threat Intelligence

```
Enterprise dark web intelligence workflow I apply in assessments:

PRE-ENGAGEMENT TI (before touching the application)
  → HIBP domain check: how many employees are in breach databases?
  → Paste site monitoring: are credentials being shared publicly?
  → Dark web forum search: is the organisation a known target?
  → GitHub secret scanning: are credentials in public repos?

CREDENTIAL EXPOSURE ASSESSMENT
  → Map all exposed employee email addresses from breach data
  → Validate sample credentials against in-scope portals (authorised)
  → Quantify exposure: "247 employee credentials are freely available"
  → Demonstrate chain: leaked creds → valid portal auth → data access

RANSOMWARE INTELLIGENCE
  → Monitor known threat actor blogs for client mentions
  → Track TTPs of relevant threat groups (MITRE ATT&CK mapping)
  → Correlate dark web chatter with active engagement findings

TOOLS & PLATFORMS
  Open source: Tor Browser, OnionSearch, Ahmia, SpiderFoot, HIBP API
  Commercial: Recorded Future, Digital Shadows, DarkOwl, Flashpoint

PROFESSIONAL STANDARDS
  ✓ Written client authorisation for all dark web investigation
  ✓ Isolated VM environment — dedicated to each engagement
  ✓ Evidence by screenshot — never download or store breach data

  ✓ Immediate notification on critical exposure discovery
```
